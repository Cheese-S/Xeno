Xeno

Introduction

This document describes a game called "Xeno", which is desigend to be a 2D Tower Defense game. The game will employ sprite-based animation, tiled backgrounds, collision detection, navmesh, physics, AI, particle effects, memory management, and real-time editing of the game world.

Technology

Xeno will be developed as an web application using Wolfie2D. WebGL is used for rendering and firebase is used for deploying.

Back Story

We are in a team of intergalactic explorers called "Xeno". As we set out to gather resrouces for our weakening colony, we landed on a seemingly unoccupied planet with everything we ever dream of. However, when the night comes, we encountered the most vicious species we have ever seen, "UMA". We have set up a cynosural field generator. All we need to do now, is to defend and hope the dawn will come.

Objective

We are playing as the commander of Xeno. Our job is to coordinate resources to defned against endless wave of enemies until reinforcements joins us. We will need to set up walls and turrets to defend against UMAs.

Gameplay

The game will work like a fixed camera, 2D top down tower defense. The user will mainly interact with the game world by placing down or selling turrets, walls, and traps.

Each wall, turrent, and trap will be upgradable by using an in game currency which will be gained by killing UMAs or building currency generating turret.

The user will need to spend his resources wisely or it will result in the destruction of cynosural field generator.

The game will be set up as follows:

  • The cynosural field generator will be located in the center of the game world.
  • Enemies will come from all side of the map.

Controls

Left Click

  • When selected nothing
    • Clicking on either a wall, trap, or turret will result in that particular object being selected
  • When selected something
    • An overlay will pop up. The user can decide whether he wants to upgrade (if upgradable) or to sell something.
  • When in edit mode (in between waves)
    • The user can select what to build by using an overlay.
    • After the user has selected something, the user can hover his mouse over the entire map to see what his base will look like after that object has been placed.
    • The user can start the next wave when they think they are ready.
  • ESC
    • Pause the game and shows the pause game menu

GUI

  • Splash Screen Features the name and the logo of our game. Click on the screen will lead a player to the home screen.

  • Home Screen A start, about, and control button will be presented. The user can click on the start button and they will be led to level selection. The user can checkout who made the game, etc. by clicking on the about button.

  • Levels Selection Screen A list of buttons that let a user to pick which level they wanna play. A user has to unlock levels (by beating the previous levels) before they can choose them from this menu.

  • Control Screen A boiled down version of controls (listed above) will be presented.

  • Help Screen This screen features the developers name + technology used. There is also a list of cheatcodes that helps developers to test out the game.

  • In Game UI

    • An indicator showing how far in the level the user is.
    • The amount of in game currency the user has.
    • Tooltip / Description about a particular object. (Only if an object is selected)
    • HP Bar for damaged actors. (?)
    • All the functionalities discussed in the Gameplay part will be provided using this UI. The UI is essentially a bar sitting at the right side of the screen. It is a menu that features multiple buttons and each button leads to their submenu.
  • In Game Menu
    • Exit
      • Quit the game
    • Controls
      • Show control screen again
    • Help
      • Show help screen again
